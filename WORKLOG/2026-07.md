# Worklog - 2026-07

## Highlights

- Connected Alexa free-text requests to Alfred Core while preserving legacy intents.
- Extracted shared Home Assistant notification delivery from domain-specific workflows.
- Extracted Giorgio speech and SSML rendering into a shared service.
- Centralized proactive notification decisions through Osvaldo.
- Routed Plex, RSVP, snoozable reminders and asynchronous laundry verification through the shared proactive path.
- Flattened the Plex update-notification workflow and removed obsolete compatibility layers.
- Clarified the responsibilities of Alfred, Giorgio, Osvaldo and Charon.
- Updated the private global project model and generated a sanitized public snapshot.
- Expanded the documented roadmap to include presence, security, cameras, energy, climate, NAS, weather and maker domains.

## Lessons Learned

- Interactive responses and proactive notifications require different policy paths.
- Speech rendering, notification policy and physical delivery should remain separate responsibilities.
- Domain services should emit events or expose tools rather than owning communication policy.
- A project model must describe the whole platform and its roadmap, not only the most recent implementation work.
- Git history replaces manual backup copies for tracked code and documentation.

## Outcome

The refactoring reduced duplicated notification and speech logic while preserving existing behavior.

Alfred remains the central tool orchestrator, Giorgio owns speech rendering, Osvaldo governs proactive communication and Charon provides media-domain intelligence.

The project documentation now reflects both the current implementation and the wider Keriol Home roadmap.
